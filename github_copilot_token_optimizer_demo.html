Every word in your prompt is a billed input token. Vague prompts also produce verbose output — the most expensive token type.

Example 1 — Refactor request ~60% fewer tokens
❌ Bloated (~180 tokens)
Hi Copilot! I have been working on this project for a while and I have this function here in my codebase. I was hoping you could take a look at it and maybe help me refactor it to make it better, cleaner, more readable, and potentially more efficient if possible. Here is the entire file for context...
✅ Tight (~70 tokens)
Refactor for readability and performance. Keep same signature. No explanation.

[selected 15-line function only]
Example 2 — Bug fix ~70% fewer tokens
❌ Vague + full file (~500 tokens)
Something is wrong with my code, it doesn't work properly when I run it. I'm not sure what the issue is. Can you look through all of this and figure out the bug? [entire 300-line file pasted]
✅ Scoped to error (~150 tokens)

Fix the null check. [10-line snippet around error]

✅ Constrained output

These are the invisible token leaks that inflate your bill before you type a single word.

📑
Open editor tabs = hidden context tokens
Every open file can be silently sent as context. 20 open tabs on a large project can add 50K–100K tokens before you ask anything.
Fix: close all unrelated tabs. Keep only the file you're actively editing.
💬
Long chat threads re-send history every turn
Each new message resends your entire conversation as input. A 20-turn chat can cost 5× more than a fresh session for the same final question.
Fix: start a new chat when switching tasks. Treat each session like a focused sticky note.
🔧
copilot-instructions.md loads on every agent step
LLM-generated boilerplate in this file adds 2K–5K tokens per step. Over 15 agent steps that's 30K–75K tokens of pure overhead.
Fix: trim to "landmines only" — constraints a good dev wouldn't guess. Delete anything obvious. Target under 300 tokens.
🔌
MCP servers cost 100–500 tokens each, every agent step
15 MCP servers × 15 agent steps = up to 112,500 tokens of pure overhead, before your actual task runs.
Fix: audit and remove MCP servers you don't use weekly. Aim for 3–5 active servers max.
📦
node_modules and generated files get indexed
Thousands of files in build folders expand repository context needlessly. They contain no useful signal for Copilot.
Fix: add node_modules/, dist/, .cache/, logs/ to .copilotignore or org content exclusion settings.
🖱️
Not selecting code forces full repository indexing
Without a selection, Copilot Chat may search the entire codebase index. Highlighting 10–20 relevant lines scopes context precisely.
Fix: always select the relevant lines before opening Copilot Chat. This alone can reduce input tokens by 80%+.
🏗️
Multi-service repos open simultaneously multiply context
Having files from multiple microservices open at once expands context scope across services without improving suggestion quality.
Fix: use VS Code workspaces or separate IDE windows per microservice. Open only the service you're working on.

These VS Code and workspace settings give you fine-grained control over what Copilot loads — and when. Add these to your .vscode/settings.json.

🚫
Disable Copilot for noisy file types
Prevents suggestions — and context loading — for XML, YAML, env files, SQL, and other low-value file types that inflate the context window.
"github.copilot.enable": {
  "*": true,
  "xml": false,
  "yaml": false,
  "env": false,
  "dotenv": false,
  "sql": false,
  "plaintext": false
}
📊
Disable workspace codebase indexing
Turns off the full semantic code search index for a specific workspace. Useful for large monorepos where indexing inflates every Chat request's context.
"github.copilot.codebase": "off"
🤖
Disable agent mode per workspace
For workspaces where you only need inline completions, disabling agents prevents accidental multi-step sessions that drain credits fast.
"chat.agent.enabled": false
🔇
Scope instructions with applyTo
Use applyTo in path-specific instruction files to load context only for relevant file types — not every file Copilot touches.
---
applyTo: "**/*.ts"
---
Use strict TypeScript.
No any types.
⏸️
Disable next edit suggestions when reading
Next Edit Suggestions are free but can trigger unnecessary background completion calls during code reading sessions. Snooze via the status bar, or disable globally.
"github.copilot.nextEditSuggestions
  .enabled": false
🎯
Commit .vscode/settings.json to the repo
Share your Copilot settings with the whole team. Teammates automatically inherit the same context exclusions, disabled file types, and workspace scope rules.
git add .vscode/settings.json
git commit -m "chore: copilot
  token optimization settings"

Set terse output as default in copilot-instructions.md
