Prompt optimization examples

Poorly worded prompts balloon input tokens. Here's how to fix them.

Example 1 — Refactor request ~60% fewer tokens
❌ Bloated (~180 tokens)
Hi Copilot! I have been working on this project for a while and I have this function here in my codebase. I was hoping you could take a look at it and maybe help me refactor it to make it better, cleaner, more readable, and potentially more efficient if possible. Here is the entire file context for you to understand what's going on...
✅ Tight (~70 tokens)
Refactor this function for readability and performance. Keep the same signature.

[selected code only]
Example 2 — Bug fix ~70% fewer tokens
❌ Vague + full file (~500 tokens)
Something is wrong with my code, it doesn't work properly when I run it. I'm not sure what the issue is. Can you look through all of this and figure out the bug? [entire 300-line file pasted]
✅ Scoped to error (~150 tokens)
TypeError: cannot read 'id' of undefined at line 42.

Fix the null check. [10-line snippet around line 42]
Example 3 — Code generation ~50% fewer output tokens
❌ Open-ended (verbose output)
Write me a function to validate email addresses in JavaScript. Include comments, examples, tests, and explain how it works.
✅ Constrained output
Write a JS email validator. Return boolean. No comments, no tests, no explanation.

Context hygiene — what secretly inflates your tokens

Open tabs = hidden input tokens. Every file open in your editor can be silently included as context. A project with 20 tabs open can add 50K–100K tokens before you type a word.
Fix: close all unrelated tabs. Only keep the file you're editing.
Long chat threads compound costs. Each new message re-sends your entire conversation history as input. A 20-turn chat can send 5x more tokens than a fresh session for the same question.
Fix: start a new chat when switching tasks. Treat sessions like sticky notes, not novels.
copilot-instructions.md is always-on context. Every agent step loads this file. LLM-generated boilerplate in it can add 2K–5K tokens per step across a 15-step agent task.
Fix: prune to only critical constraints ("landmines"). Delete anything a good developer already knows.
MCP servers cost 100–500 tokens each, per agent step. 15 MCP servers × 15 agent steps = up to 112,500 tokens of pure overhead — before your actual task even runs.
Fix: audit and remove MCP servers you don't actively use. Keep only the ones you touch weekly.
node_modules and generated files get indexed. These folders have thousands of files that expand repository context needlessly.
Fix: add content exclusion rules in your .copilotignore or org settings for node_modules, dist, .cache, logs.
Select code before asking. Without a selection, Copilot Chat may index the whole repo. A 20-line selection scopes the context to just that code.
Fix: always highlight the relevant lines first, then open Copilot Chat.
