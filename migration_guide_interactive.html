Setup — 5 minutes

Everything you need before typing the first migration command.

Prerequisites

  • VS Code installed
  • Claude Code extension installed
  • .NET 8 SDK — run dotnet --version must show 8.x
  • Git access to your repo
  • Artifactory NuGet feed URL from your org

Step 1 — Clone and add this kit

$git clone https://your-org/your-aspnet-app.git
$cp -r /path/to/migration-kit/.github ./your-aspnet-app/
$code ./your-aspnet-app

Step 2 — Configure Artifactory (one line)

Edit .github/nuget.config — replace the placeholder URL with your org's real Artifactory URL:

nuget.config
- value="https://artifactory.yourorg.com/artifactory/api/nuget/nuget-virtual"
+ value="https://YOUR-REAL-ARTIFACTORY-URL/artifactory/api/nuget/nuget-virtual"
Never commit ARTIFACTORY_USER or ARTIFACTORY_TOKEN — add them as environment variables or GitHub secrets.

Step 3 — Start the migration

In VS Code, open Claude Code and type exactly this:

@claude start migration

Claude reads CLAUDE.md (~200 tokens), scans your solution, builds the map, creates the Core project structure, and begins migrating in dependency order.

ADO.NET — migration path

Two options: Dapper (minimal change) or EF Core (full ORM). Dapper preferred when you want to keep stored procs.

ADO.NET SqlCommand
→ Dapper (keep SQL)
Add Dapper package. Replace SqlDataReader loops with QueryAsync<T>. Stored procs work unchanged. Fastest migration.
ADO.NET raw SQL
→ EF Core (full ORM)
Scaffold DbContext from existing DB. Replace all raw SQL with LINQ. Best for long-term maintainability.

Apigee / external API calls

Replace RestSharp / WebClient with IHttpClientFactory typed clients. API keys in Key Vault.

Typed client pattern

// Program.cs
builder.Services.AddHttpClient<IApigeeProductClient, ApigeeProductClient>(c => {
    c.BaseAddress = new Uri(builder.Configuration["Apigee:BaseUrl"]!);
    c.Timeout = TimeSpan.FromSeconds(30);
});

// ApigeeProductClient.cs
public class ApigeeProductClient(HttpClient http, IConfiguration config)
    : IApigeeProductClient
{
    public async Task<List<Product>> GetProductsAsync(CancellationToken ct = default)
    {
        using var req = new HttpRequestMessage(HttpMethod.Get, "/v1/products");
        req.Headers.Add("x-api-key", config["Apigee:ApiKey"]);
        var resp = await http.SendAsync(req, ct);
        resp.EnsureSuccessStatusCode();
        return await resp.Content.ReadFromJsonAsync<List<Product>>(ct) ?? [];
    }
}
Store Apigee:ApiKey and Apigee:BaseUrl in Azure Key Vault, not appsettings.json. Add builder.Configuration.AddAzureKeyVault(...) in Program.cs.

Ping / OIDC login migration

Replace FormsAuthentication with OpenID Connect middleware. Ping acts as the Identity Provider (IdP).

Program.cs OIDC setup

builder.Services
    .AddAuthentication(options => {
        options.DefaultScheme = CookieAuthenticationDefaults.AuthenticationScheme;
        options.DefaultChallengeScheme = OpenIdConnectDefaults.AuthenticationScheme;
    })
    .AddCookie(options => {
        options.Cookie.HttpOnly = true;
        options.Cookie.SecurePolicy = CookieSecurePolicy.Always;
        options.Cookie.SameSite = SameSiteMode.Strict;
        options.ExpireTimeSpan = TimeSpan.FromHours(8);
    })
    .AddOpenIdConnect(options => {
        options.Authority = config["Ping:DiscoveryUrl"];   // from Key Vault
        options.ClientId = config["Ping:ClientId"];        // from Key Vault
        options.ClientSecret = config["Ping:ClientSecret"];// from Key Vault
        options.ResponseType = OpenIdConnectResponseType.Code;
        options.Scope.Add("openid");
        options.Scope.Add("profile");
        options.Scope.Add("email");
        options.SaveTokens = true;
        options.GetClaimsFromUserInfoEndpoint = true;
    });

app.UseAuthentication();
app.UseAuthorization();
Manual step: get Ping:DiscoveryUrl, Ping:ClientId, Ping:ClientSecret from your identity team. Add to Azure Key Vault before this code can work.

Crystal Reports — decision required

No .NET Core support. Files flagged as 🚧 BLOCK in CODEBASE-MAP.md. A human decision is needed before migration can continue.

Crystal Reports has no official .NET Core equivalent. Choose one of the four options below.

Option 1 — Microsoft.Reporting.NETCore (free, simple reports)

<PackageReference Include="Microsoft.Reporting.NETCore" Version="3.0.0" />

Renders existing .rdlc report files. Works for local reports — no server deployment. Best for simple tabular reports. Zero cost.

Option 2 — FastReport Community Edition (free, complex reports)

<PackageReference Include="FastReport.OpenSource" Version="2024.1.0" />

Full report designer, charts, sub-reports. Open source. Reports need redrawing from scratch in FastReport format.

Option 3 — SSRS (SQL Server Reporting Services)

If your org already has SQL Server licensed. Reports stay in SSRS, app calls SSRS REST API. No package needed.

Option 4 — Telerik Reporting or DevExpress (paid)

Full-featured, closest to Crystal Reports feature set. Requires license purchase. Reports need redrawing.

After selecting an option, update CODEBASE-MAP.md: change 🚧 BLOCK to ⏳ QUEUE for all report files, add the chosen package to the approved registry.

Security checks — Artifactory-only

No external tools. All security checking uses .NET SDK built-ins and your Artifactory NuGet analyzer package.

CVE scan (built into .NET SDK)

$dotnet list package --vulnerable --include-transitive
Run after every .csproj change. CVSS ≥ 7.0 (HIGH) = block PR. Built into the SDK — no Artifactory lookup needed.

CI pipeline — Artifactory-only

Self-hosted runners with Artifactory access. No external actions, no npm/pip installs.

.github/workflows/migration-ci.yml — key jobs

jobs:
  build:
    runs-on: [self-hosted]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-dotnet@v4
        with: { dotnet-version: '8.0.x' }

      - name: Configure Artifactory (no nuget.org)
        run: |
          dotnet nuget remove source nuget.org 2>/dev/null || true
          dotnet nuget add source "$ARTIFACTORY_URL/api/nuget/nuget-virtual" \
            --name Artifactory \
            --username "$ARTIFACTORY_USER" \
            --password "$ARTIFACTORY_TOKEN" \
            --store-password-in-clear-text

      - name: Restore (locked mode)
        run: dotnet restore --locked-mode

      - name: Build + Roslyn/Org analyzers
        run: dotnet build --no-restore --configuration Release

      - name: CVE scan (SDK built-in)
        run: |
          dotnet list package --vulnerable --include-transitive | tee vuln.txt
          grep -qi "High\|Critical" vuln.txt && exit 1 || true

      - name: Secret scan (grep-based, no pip)
        run: |
          git diff origin/main...HEAD -- '*.cs' '*.json' '*.config' \
            | grep "^+" | grep -iv "^+++" \
            | grep -iE '(password\s*=\s*[^$<{"'"'"'][^;]{3,})' \
            | grep -iv '(//|todo|example|placeholder)' \
            && echo "SECRETS FOUND" && exit 1 || true

      - name: Unit tests + coverage
        run: |
          dotnet test --no-build --configuration Release \
            --filter "Category!=Integration" \
            --collect:"XPlat Code Coverage"

      - name: Framework dependency gate
        run: |
          HITS=$(find src-core/ -name "*.cs" -not -path "*/obj/*" \
            | xargs grep -l "using System\.Web" 2>/dev/null || true)
          [ -n "$HITS" ] && echo "$HITS" && exit 1 || echo "Clean"
